# Gemini CLI Project Configuration

## Mission: AutoClaw Management & Assistance
One of the primary roles for Gemini CLI in this workspace is to manage, edit, and enhance **AutoClaw** (an OpenClaw distribution). This includes:
*   Refining the agent's logic and persona via workspace Markdown files.
*   Developing and integrating new features or "skills."
*   Managing AutoClaw's configuration and local environment.
*   Troubleshooting and optimizing its event-driven architecture.

## Essential Documentation
*   [AutoClaw Architecture Overview](./AUTOCLAW_ARCHITECTURE.md): Detailed findings on AutoClaw/OpenClaw's structure, 5-layer system, and local Windows implementation.

## Mandates
- **Architecture Awareness:** Always refer to the [AutoClaw Architecture Overview](./AUTOCLAW_ARCHITECTURE.md) before making structural changes to AutoClaw.
- **Surgical Updates:** When modifying AutoClaw's workspace Markdown files (`SOUL.md`, `IDENTITY.md`, etc.), ensure the "vibe" and core principles remain consistent unless explicitly instructed to change them.
- **Verification:** Any changes to AutoClaw skills or configuration should be verified against the `openclaw.json` settings and the local skill structure.
